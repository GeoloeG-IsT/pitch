## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, card internal gutters |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section card padding, TOC item spacing |
| xl | 32px | Gap between section cards |
| 2xl | 48px | Document group separators |
| 3xl | 64px | Page top/bottom padding, viewer container margins |

Exceptions: FAB touch target is 56px (48px icon area + 4px padding each side) for mobile accessibility compliance (WCAG 2.5.5 target size).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.6 |
| Label | 14px | 500 (medium) | 1.4 |
| Heading | 24px | 600 (semibold) | 1.25 |
| Display | 32px | 700 (bold) | 1.2 |

Notes:
- Body at 16px/1.6 optimized for long-form reading (pitch content is the primary artifact).
- Display is used exclusively for document group titles (e.g., "Pitch Deck", "Financial Model").
- Heading is used for section card titles derived from heading chunks.
- Label is used for metadata text (section context chip, TOC items, Q&A timestamps, citation references).
- Key metric numbers in financial cards use Display size at weight 700.

## Color

Uses existing shadcn neutral theme tokens from `globals.css`. No custom colors introduced.

| Role | Token | Value (light) | Usage |
|------|-------|---------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) / hsl(0 0% 100%) | Page background behind cards |
| Secondary (30%) | `--card` | oklch(1 0 0) / hsl(0 0% 100%) | Section cards, Q&A panel, TOC sidebar |
| Surface tint | `--muted` | oklch(0.97 0 0) / hsl(240 4.8% 95.9%) | Viewer page background (subtle off-white behind white cards), Q&A message bubbles for AI responses |
| Accent (10%) | `--primary` | oklch(0.205 0 0) / hsl(240 5.9% 10%) | See reserved list below |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) / hsl(0 84.2% 60.2%) | Error states only |

Accent (`--primary`) reserved for:
- FAB button background
- Active TOC item indicator (left border or background)
- "Ask a question" submit button in Q&A panel
- Section context chip background
- Clickable citation links (hover state)

Viewer page background: Use `bg-muted` (the subtle off-white) so white `bg-card` section cards visually lift off the surface. This creates the Gamma-style card-on-surface depth without additional shadows.

## Layout Contract

### Desktop (>=1024px)

```
+----------------------------------------------------------+
| [Logo] Zeee Pitch Zooo          [company name]           |  <- sticky header, h-14
+----------+-----------------------------------------------+
|          |                                                |
|   TOC    |   [Document Group Title: "Pitch Deck"]         |
|  240px   |                                                |
|  sidebar |   +------------------------------------------+ |
|          |   | Section Card (heading)                    | |
|  fixed   |   +------------------------------------------+ |
|  left    |   | Section Card (text - markdown)            | |
|          |   +------------------------------------------+ |
|          |   | Section Card (table - metric grid)        | |
|          |   +------------------------------------------+ |
|          |                                                |
|          |   [Document Group Title: "Financial Model"]    |
|          |   +------------------------------------------+ |
|          |   | Section Card (table - metric grid)        | |
|          |   +------------------------------------------+ |
|          |                                                |
+----------+-----------------------------------------------+
                                                      [FAB] <- bottom-right, 56px
```

- Content column: `max-w-5xl` (1024px), centered in remaining space after sidebar
- TOC sidebar: 240px fixed width, `bg-card` with `border-r`
- Cards: full width within content column, `bg-card` on `bg-muted` page background
- Card gap: 32px (xl) between cards, 48px (2xl) between document groups

### Mobile (<768px)

- TOC sidebar hidden; hamburger icon in header opens slide-out drawer
- Section cards: full-width with 16px (md) horizontal page padding
- FAB: bottom-center, 56px, 16px from bottom edge
- Q&A panel: full-screen takeover via Dialog (not Sheet) with back button in header
- Metric cards: stack vertically, full-width (no grid)

### Tablet (768px - 1023px)

- TOC sidebar collapsed by default; toggle button in header
- Content column: `max-w-3xl` (768px) centered
- Q&A panel: Sheet from right, 400px width
- Metric cards: 2-column grid

## Interaction Contract

### Scroll Behavior

- Smooth scrolling (`scroll-behavior: smooth`) on the content area
- TOC highlights the currently visible section using Intersection Observer (threshold: 0.3)
- Clicking a TOC item scrolls the corresponding section into view with 80px offset (clears sticky header)
- Citation clicks in Q&A panel scroll the viewer to the referenced section and briefly highlight it (2s ring pulse)

### FAB Behavior

- Default state: Circular button with MessageSquare icon (Lucide `message-square`)
- Hover: Tooltip "Ask a question"
- Click: Opens Q&A Sheet panel (desktop/tablet) or full-screen Dialog (mobile)
- While Q&A panel is open: FAB hides (panel has its own close affordance)
- Section detection: FAB tracks which section is currently in the viewport center and passes that context to the Q&A panel

### Q&A Panel Behavior

- Opens pre-scoped to the section visible when FAB was clicked
- Section context chip displayed above input: Badge with section name + X dismiss button
- Dismissing chip switches to global Q&A mode (no section scoping)
- Conversation thread: scrollable area showing all Q&A pairs from current session
- Each answer streams in real-time via existing WebSocket infrastructure
- Citations below each answer are clickable; clicking scrolls viewer to that section
- Input: single-line text input with submit button (Enter key or click)
- Loading state: pulsing dots indicator below the last user message while streaming

### TOC Sidebar Behavior

- Groups sections by source document (Collapsible per document)
- Each section item: clickable, truncated text with Tooltip on hover
- Active section: `bg-accent` background + `border-l-2 border-primary` left indicator
- Mobile drawer: slide-out from left, overlay with backdrop, close on item click or backdrop click

### Loading States

- Initial load: Skeleton cards (3 stacked) mimicking section card dimensions
- Each skeleton: rounded rectangle matching card aspect ratio, with inner line skeletons for text
- TOC: 6 skeleton lines at label size
- Transition: fade-in (200ms opacity) when real content replaces skeletons

### Error States

- Failed to load pitch content: full-page centered error card
- Failed Q&A query: inline error message below the failed question in thread
- WebSocket disconnection: muted banner at top of Q&A panel with reconnect button

## Section Rendering Rules

### chunk_type: "text"

- Render as rich Markdown inside Card body
- Support: bold, italic, lists (ordered/unordered), links, inline code, code blocks
- Use `prose` class or equivalent for consistent typography
- Max content width within card: readable line length (~75ch)

### chunk_type: "table"

- Render as MetricCard grid inside Card body
- Parse table content to extract label-value pairs
- Grid: 3 columns on desktop, 2 on tablet, 1 on mobile
- Each MetricCard: Label (14px, muted-foreground), Value (32px, bold, foreground), optional sub-label
- If table data cannot be parsed into key-value pairs, fall back to a styled HTML table with `border-collapse`, alternating row backgrounds using `bg-muted`

### chunk_type: "heading"

- Render as a section divider, not inside a Card
- Heading text at 24px/600 weight
- Subtle horizontal rule (`Separator`) below the heading text
- 48px (2xl) top margin to separate from previous document group content

### chunk_type: "image_caption"

- Render inside a Card with `bg-muted` background (visually distinct from text cards)
- Small Image icon (Lucide `image`) at 16px, muted-foreground, inline before caption text
- Caption text in italic, 14px, muted-foreground
- Compact padding: 16px (md) all sides
