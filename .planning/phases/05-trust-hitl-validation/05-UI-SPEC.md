## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.6 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 24px | 600 (semibold) | 1.25 |
| Display | 32px | 600 (semibold) | 1.2 |

Notes:
- Two weights only: 400 (regular) for Body and Label, 600 (semibold) for Heading and Display.
- Display is used exclusively for the dashboard page title ("Validation Dashboard").
- Heading is used for tab section titles ("Pending Review", "History").
- Body is used for question text, AI draft answer text, and founder-edited answer text in review cards.
- Label is used for confidence tier labels, timestamps, section context references, and review status tags.

## Color

Uses existing shadcn neutral theme tokens from `globals.css`. Three new semantic colors introduced for confidence tiers via custom CSS variables.

| Role | Token | Value (light) | Usage |
|------|-------|---------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) | Page background |
| Secondary (30%) | `--card` | oklch(1 0 0) | Review cards, tab panels |
| Surface tint | `--muted` | oklch(0.97 0 0) | Dashboard page background, history card muted state |
| Accent (10%) | `--primary` | oklch(0.205 0 0) | See reserved list below |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) | Reject action button, low-confidence badge |

### Confidence Tier Colors (new CSS variables)

| Token | Value (light) | Value (dark) | Usage |
|-------|---------------|--------------|-------|
| `--color-confidence-high` | `hsl(142 71% 45%)` | `hsl(142 71% 35%)` | High confidence badge background (green) |
| `--color-confidence-moderate` | `hsl(38 92% 50%)` | `hsl(38 92% 40%)` | Moderate confidence badge background (yellow/amber) |
| `--color-confidence-low` | `hsl(0 84% 60%)` | `hsl(0 63% 31%)` | Low confidence badge background (red), maps to --destructive |

Each confidence badge uses white foreground text on its tier color background.

### Verified Badge

| Token | Value | Usage |
|-------|-------|-------|
| `--color-verified` | `hsl(142 71% 45%)` | "Verified" badge for founder-reviewed answers (same green as high confidence) |

Accent (`--primary`) reserved for:
- "Approve" action button
- Active tab indicator
- Pending review count badge in navigation
- Inline edit cursor/focus ring
- "Submit" button for replacement answer

**Focal point:** Primary visual anchor is the review card queue; the Approve button is the primary interactive focal point, enabling fast throughput.

## Interaction Contract

### Confidence Badge Behavior

- Default state: Colored badge with tier label text ("High confidence", "Moderate confidence", "Low confidence")
- Hover/tap: Tooltip reveals numeric percentage (e.g., "Score: 72%")
- Badge uses the tier color as background with white foreground text
- "Verified" badge: green background with ShieldCheck icon (Lucide `shield-check`), text "Verified"

### Dashboard Tab Behavior

- Two tabs: "Pending Review" (shows count) and "History"
- "Pending Review" tab label includes live count badge: "Pending Review (3)"
- Default active tab: "Pending Review" if count > 0, otherwise "History"
- Tab content area uses ScrollArea for vertical overflow

### Review Card Behavior

- Static display: question, answer text, confidence badge, citations, section context, timestamp
- "Approve" button: publishes answer as-is, card animates out of Pending queue, appears in History with "Approved" status tag
- "Edit" button: answer text transforms into Textarea (InlineEditor), Approve/Cancel buttons replace action row
- "Reject" button: opens RejectionForm below the card (Textarea for replacement answer + Submit button)
- After any action: card transitions to History tab with appropriate status

### Inline Edit Behavior

- Click "Edit": answer text becomes an auto-focused Textarea with the current answer pre-filled
- Textarea height: auto-resizing, min 3 rows
- Action row changes to: "Save & Approve" (primary) + "Cancel" (ghost)
- "Save & Approve": submits edited text, publishes to investor
- "Cancel": reverts to read-only view, restores original action buttons

### Rejection Behavior

- Click "Reject": RejectionForm appears below the card with empty Textarea
- Placeholder text: "Write a replacement answer for the investor..."
- "Submit Replacement" button (primary) is disabled until Textarea has content
- On submit: replacement answer publishes to investor with "Verified" badge
- Investor never sees a dead end -- rejection always produces a founder-written answer

### Investor Push Behavior

- When founder approves/edits/rejects a queued answer, the "being verified" placeholder smoothly animates into the approved/replacement answer
- Animation: fade-out placeholder (200ms), fade-in answer with badge (300ms)
- WebSocket message type delivers the approved answer payload to active investor sessions
- Founder-reviewed answers display "Verified" badge regardless of original confidence tier

### Pending Count Badge

- Displays in the site header/navigation area
- Shows numeric count of pending review items
- Hidden when count is 0
- Updates in real-time as new low-confidence answers enter the queue

### Loading States

- Dashboard initial load: 3 skeleton cards matching ReviewCard dimensions
- Each skeleton: card shape with 3 line skeletons (question, answer, action row)
- Tab count shows "--" while loading
- Transition: fade-in (200ms opacity) when real content replaces skeletons

### Error States

- Failed to load review queue: centered error card within tab panel
- Failed to submit review action: toast notification with error message, card reverts to pre-action state
- WebSocket push failure: silent retry (investor can refresh to see approved answer)
