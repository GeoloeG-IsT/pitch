## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.6 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 24px | 600 (semibold) | 1.25 |
| Display | 32px | 600 (semibold) | 1.2 |

Notes:
- Two weights only: 400 (regular) for Body and Label, 600 (semibold) for Heading and Display.
- Display is used exclusively for the presenter view page title ("Live Q&A").
- Heading is not used in this phase -- the presenter view is a flat question feed with no section headings.
- Body is used for question text and AI draft answer text in presenter question cards.
- Label is used for investor identity labels, timestamps, action button labels, connected investor count, LIVE banner text, and dismissal messages.

## Color

Uses existing shadcn neutral theme tokens from `globals.css`. One new semantic color introduced for the LIVE indicator.

| Role | Token | Value (light) | Usage |
|------|-------|---------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) | Presenter page background, pitch viewer background |
| Secondary (30%) | `--card` | oklch(1 0 0) | Question cards in presenter feed |
| Surface tint | `--muted` | oklch(0.97 0 0) | Presenter page background tint, draft answer area |
| Accent (10%) | `--primary` | oklch(0.205 0 0) | See reserved list below |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) | Dismiss action button |

### Live Session Color (new CSS variable)

| Token | Value (light) | Value (dark) | Usage |
|-------|---------------|--------------|-------|
| `--color-live` | `hsl(0 84% 60%)` | `hsl(0 63% 50%)` | LIVE banner background, LIVE dot pulse, Go Live active state |

The LIVE color uses red (matching broadcast/recording convention) to signal an active, real-time session state.

Accent (`--primary`) reserved for:
- "Approve" action button on presenter question cards
- "Go Live" toggle button (inactive state uses outline variant)
- Connected investor count in presenter header
- Override editor submit button ("Publish Answer")
- Edit save button ("Save & Publish")

**Focal point:** Primary visual anchor is the question feed; the Approve button is the primary interactive focal point, enabling fast question throughput during a live session.

## Interaction Contract

### Go Live Flow

1. Founder clicks "Go Live" button on dashboard header
2. Confirmation Dialog appears: "Start a live session? Investors will see a LIVE indicator and all questions will route through you for approval."
3. Two actions: "Start Live Session" (primary) and "Cancel" (ghost)
4. On confirm: API call POST /api/v1/sessions, button changes to "Live Now" with red dot, toast "Live session started"
5. Founder can navigate to /present (linked from the "Live Now" button or a separate "Open Presenter View" link)

### End Session Flow

1. From presenter view: founder clicks "End Session" button in header
2. AlertDialog appears: "End this live session? Unanswered questions will move to your review queue."
3. Two actions: "End Session" (destructive) and "Keep Live" (outline)
4. On confirm: API call DELETE /api/v1/sessions/{id}, LIVE banner disappears for investors, presenter view shows "Session ended" message, toast "Live session ended"
5. Dashboard "Go Live" button reverts to inactive state

### Question Card Behavior

- New questions appear at top of feed with slide-down animation (300ms)
- While AI draft is generating: skeleton pulse in the answer area, action buttons disabled
- Once draft is ready: answer text fades in (200ms), action buttons become active
- **Approve**: publishes AI draft as-is to investor, card animates out (fade + collapse, 300ms)
- **Edit**: answer text transforms into Textarea (InlineEditor), action row changes to "Save & Publish" (primary) + "Cancel" (ghost)
- **Override**: AI draft area clears, replaced by empty Textarea (OverrideEditor) with placeholder "Write your answer...", action row changes to "Publish Answer" (primary) + "Cancel" (ghost)
- **Dismiss**: card animates out (fade + collapse, 300ms), investor sees dismissal message, toast "Question dismissed"

### Investor Live Experience

- When session starts: LIVE banner slides down (300ms) at top of pitch viewer
- Q&A panel works identically to async mode (FAB + slide-in panel)
- When investor submits a question: "being reviewed" placeholder appears instead of streaming answer
- When founder approves/edits/overrides: placeholder transitions to the published answer (fade-out 200ms, fade-in 300ms)
- When founder dismisses: placeholder transitions to dismissal message
- When session ends: LIVE banner slides up and disappears (300ms), toast "Live session ended. You can continue exploring the pitch.", Q&A reverts to async mode

### Connected Investor Count

- Displayed in presenter header: "{N} investor{s} connected"
- Click/hover reveals DropdownMenu listing investor identities (email addresses)
- Count updates in real-time as investors connect/disconnect
- Deduplicates by investor identity (multiple tabs from same investor count as one)

### Loading States

- Presenter initial load: 2 skeleton cards matching QuestionCard dimensions
- Each skeleton: card shape with 3 line skeletons (investor label, question, answer area)
- Transition: fade-in (200ms opacity) when real content replaces skeletons

### Error States

- Failed to start session: toast "Could not start live session. Please try again."
- Failed to end session: toast "Could not end live session. Please try again."
- Failed to approve/edit/override/dismiss: toast "Could not publish answer. Please try again.", card reverts to pre-action state
- WebSocket disconnection during live session: toast "Connection lost. Reconnecting...", auto-reconnect attempt
