# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |
| Preset | default (no custom preset) |
| Component library | Radix UI (via shadcn) |
| Icon library | lucide-react |
| Font | Inter (system fallback: ui-sans-serif, system-ui, sans-serif) |

**Initialization command:**
```bash
npx shadcn@latest init
```

Accept defaults: New York style, CSS variables enabled, Tailwind v4, `@/components/ui` path.

**shadcn components needed for Phase 1:**
```bash
npx shadcn@latest add badge card
```

## Color

Uses shadcn default theme (CSS variables in HSL). Specific values below are the shadcn "New York" defaults in light mode.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (hsl 0 0% 100%) | Page background |
| Secondary (30%) | `--card` (hsl 0 0% 100%) / `--muted` (hsl 240 4.8% 95.9%) | Status cards, muted labels |
| Accent (10%) | `--primary` (hsl 240 5.9% 10%) | Status badge active state, page title |
| Destructive | `--destructive` (hsl 0 84.2% 60.2%) | Error/unreachable status badge only |

Accent reserved for: page title text, "healthy" status badge background.

**Semantic status colors (specific to health page):**

| Status | Color token | Rendered as |
|--------|-------------|-------------|
| OK / healthy | green-500 (`#22c55e`) | Badge variant: green background, white text |
| Degraded | yellow-500 (`#eab308`) | Badge variant: yellow background, black text |
| Unreachable / error | destructive (`--destructive`) | Badge variant: red background, white text |

These status colors use Tailwind utility classes (e.g., `bg-green-500 text-white`), not custom CSS variables. They are scoped to the health page and do not pollute the design system.

| Footer alignment | Center |
| Footer style | Label size (14px), muted foreground color |

### Data Flow

The page fetches `/api/health` on load (client-side) and displays results. No polling, no WebSocket. Manual refresh via browser reload.

| Service | Check method | Response field |
|---------|-------------|----------------|
| Frontend | Implicit (page rendered) | Always "Healthy" |
| Backend API | GET `/api/v1/health` via Next.js rewrite proxy | `backend` field from health route |
| Supabase | Backend health endpoint checks Supabase connectivity | `database` field from health route |

### States

| State | What renders |
|-------|-------------|
| Loading | Three cards with skeleton pulse animation (shadcn default) |
| All healthy | Three cards with green badges, summary: "All systems operational" |
| Backend down | Frontend green, Backend red ("Unreachable"), Supabase yellow ("Degraded"), summary: degraded message |
| Supabase down | Frontend green, Backend green, Supabase red ("Unreachable"), summary: degraded message |
| All down | Frontend green (always), Backend red, Supabase red, summary: error message |
