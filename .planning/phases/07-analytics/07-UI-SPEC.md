## Layout Contract

### Analytics Tab Structure

```
[Dashboard Page - max-w-4xl mx-auto py-8 px-4]
  [Tabs]
    [TabsList]
      Reviews | Access | Analytics {AnalyticsCountBadge}
    [TabsContent value="analytics" className="mt-6"]
      [SectionHeatmap Card]        -- aggregate attention across all investors
        CardHeader: "Section Attention"
        CardContent: horizontal bar segments, one per pitch section
      [InvestorTable Card]          -- below heatmap
        CardHeader: "Investor Engagement"
        CardContent:
          Table columns: Investor | Last Viewed | Time Spent | Questions | Engagement
          High-engagement rows sorted to top
          Each row is Collapsible:
            CollapsibleTrigger: the table row (ChevronRight icon rotates on open)
            CollapsibleContent: InvestorDetailRow
```

### Section Heatmap Layout

- Single horizontal bar, divided into segments proportional to section content length
- Each segment colored by aggregate time spent (all investors combined)
- Section name labels below the bar in 12px muted-foreground
- Bar height: 32px with border-radius-sm
- Tooltip on hover showing section name + total time (use native title attribute for PoC)

### Investor Detail Row (expanded)

```
[CollapsibleContent className="bg-muted/50 px-6 py-4"]
  [Grid: 2 columns on md+, 1 column on mobile]
    [Left: Per-Section Breakdown]
      Section name (14px semibold) | time bar (proportional width) | duration label
      ... repeat for each section
    [Right: Question Log]
      Heading: "Questions Asked" (14px semibold)
      List of questions with timestamps (12px muted-foreground)
      Each question: text (14px) + "Asked {relative time}" label
```

### Engagement Badge Variants

| Level | Label | Style |
|-------|-------|-------|
| Hot | "Hot" | `bg-[hsl(12,76%,61%)] text-white` + Flame icon (lucide `Flame`) |
| Active | "Active" | `variant="secondary"` (default Badge) |
| Viewed | "Viewed" | `variant="outline"` (muted treatment) |

Rules (from CONTEXT.md):
- **Hot**: ANY of: 5+ min on financials, 3+ deep questions, 2+ return visits, 100% scroll completion
- **Active**: Viewed within last 7 days but no hot signals
- **Viewed**: Viewed but inactive for 7+ days

## Interaction Contract

### Expandable Table Rows

- Click anywhere on a table row to toggle expanded detail
- ChevronRight icon in first column rotates 90 degrees clockwise when expanded
- Transition: `transition-transform duration-200`
- Only one row expanded at a time (accordion behavior) -- keeps context manageable
- Collapsible uses `forceMount` to avoid layout jumps

### Heatmap Hover

- Cursor changes to pointer on heatmap segments
- Tooltip appears showing: "{Section Name}: {total minutes}m across {N} investors"
- No click action on heatmap (display-only for PoC)

### Real-time Notification Toast

- Sonner toast appears when a new investor opens the pitch for the first time
- Toast content: "{Investor email or 'An investor'} just opened your pitch"
- Toast duration: 5 seconds (Sonner default)
- Delivered via existing WebSocket infrastructure to founder clients

### Analytics Count Badge

- Polls every 30 seconds (matches PendingCountBadge pattern)
- Shows count of "new views since last dashboard visit"
- Clears when founder switches to Analytics tab (marks events as seen via API call)
- Badge renders inside TabsTrigger for "Analytics", not on SiteNav

### Table Sorting

- High-engagement ("Hot") investors always sort to top
- Within engagement tiers, sort by last viewed descending (most recent first)
- No user-clickable sort controls for PoC -- single fixed sort order

## State Contract

### Loading State

- Skeleton placeholders matching existing dashboard pattern:
  - Heatmap card: single `Skeleton className="h-[32px] w-full"` inside card
  - Table: 3x `Skeleton className="h-10 w-full"` stacked with `space-y-3`

### Empty State

- Displayed when no investors have viewed the pitch yet
- Appears inside the InvestorTable card area (heatmap card hidden when no data)

### Error State

- API fetch failure shows inline error within the Analytics tab content
- Does not affect other dashboard tabs

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Expandable rows | `aria-expanded` on trigger, `aria-controls` pointing to detail panel id |
| Engagement badges | Badge text is the accessible label (no icon-only badges) |
| Heatmap segments | `role="img"` on container, `aria-label="Section attention heatmap"` |
| Heatmap segment | `title` attribute with section name and time for tooltip |
| Color contrast | All badge text meets WCAG AA (4.5:1 minimum) -- white text on chart-1 orange passes |
| Toast notifications | Sonner uses `role="status"` by default (live region) |
