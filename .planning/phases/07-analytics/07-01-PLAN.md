Create migration file `supabase/migrations/00007_analytics_events.sql` with:

```sql
CREATE TABLE public.analytics_events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  founder_id UUID NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
  user_id UUID REFERENCES public.users(id) ON DELETE SET NULL,
  share_token_id UUID REFERENCES public.share_tokens(id) ON DELETE SET NULL,
  session_id TEXT NOT NULL,
  event_type TEXT NOT NULL CHECK (event_type IN ('page_open', 'page_close', 'section_time', 'scroll_depth')),
  section_id TEXT,
  duration_ms INTEGER,
  scroll_depth INTEGER CHECK (scroll_depth IS NULL OR scroll_depth IN (25, 50, 75, 100)),
  metadata JSONB DEFAULT '{}',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  CONSTRAINT identity_required CHECK (user_id IS NOT NULL OR share_token_id IS NOT NULL)
);

CREATE INDEX idx_analytics_founder ON analytics_events(founder_id);
CREATE INDEX idx_analytics_session ON analytics_events(session_id);
CREATE INDEX idx_analytics_user ON analytics_events(user_id) WHERE user_id IS NOT NULL;
CREATE INDEX idx_analytics_token ON analytics_events(share_token_id) WHERE share_token_id IS NOT NULL;
CREATE INDEX idx_analytics_created ON analytics_events(created_at DESC);

ALTER TABLE public.analytics_events ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Founders read own analytics"
  ON public.analytics_events FOR SELECT
  USING (auth.uid() = founder_id);
```

from pydantic import BaseModel

class SectionTimeEvent(BaseModel):
    section_id: str
    duration_ms: int

class BeaconPayload(BaseModel):
    session_id: str
    user_id: str | None = None
    share_token_id: str | None = None
    founder_id: str
    events: list[SectionTimeEvent]
    scroll_depth: int | None = None

    session_end: int    # Unix timestamp ms

class InvestorSummary(BaseModel):
    investor_key: str  # user_id or share_token_id
    investor_label: str  # email or "Anonymous (...last4)"
    last_viewed: str | None = None
    total_time_ms: int = 0
    question_count: int = 0
    session_count: int = 0
    max_scroll_depth: int = 0
    financials_time_ms: int = 0
    engagement: str = "viewed"  # "hot" | "active" | "viewed"

class SectionTime(BaseModel):
    section_id: str
    duration_ms: int

class QuestionEntry(BaseModel):
    question: str
    created_at: str

class InvestorDetail(BaseModel):
    investor_key: str
    sections: list[SectionTime]
    questions: list[QuestionEntry]

class AnalyticsSummaryResponse(BaseModel):
    investors: list[InvestorSummary]

class NewViewCountResponse(BaseModel):
    count: int

Create `apps/api/app/api/v1/analytics.py` with these endpoints:

1. **POST /analytics/events** (status 204, no auth) -- sendBeacon ingestion endpoint:
   - Parse raw request body via `request.body()` then `BeaconPayload.model_validate_json(body)` (sendBeacon may send text/plain)
   - Build rows list: one `page_open` event (with session_start in metadata), one per section_time event, one `scroll_depth` event if scroll_depth is set, one `page_close` event (with session_end in metadata)
   - Insert all rows via `get_service_client().table("analytics_events").insert(rows).execute()`
   - After insert, check if this is a first-view: query analytics_events for `page_open` events matching this founder_id + (user_id or share_token_id), count > 1 means NOT first view
   - If first view AND share_token_id is present: look up share_token row to get investor_email, then call `await notify_founder(founder_id, {"type": "pitch_opened", "investor": investor_email or "An investor", "share_token_id": share_token_id})`
   - Return `Response(status_code=204)`

2. **GET /analytics/summary** (requires `get_current_user`) -- returns `AnalyticsSummaryResponse`:
   - Query analytics_events grouped by investor identity: `SELECT COALESCE(user_id::text, share_token_id::text) as investor_key, ...` aggregating MAX(created_at) as last_viewed, SUM(duration_ms) for section_time events as total_time, COUNT(DISTINCT session_id) as session_count, MAX(scroll_depth) as max_scroll_depth
   - Join with share_tokens to get investor_email
   - Query queries table grouped by the same investor_key to get question counts
   - Compute financials_time: SUM(duration_ms) WHERE section_id matches sections from documents with file_type='xlsx' (query documents table for xlsx doc IDs, then get chunk IDs from chunks table)
   - Compute engagement tier per investor: "hot" if ANY of: financials_time_ms >= 300000 (5 min), question_count >= 3, session_count >= 2, max_scroll_depth >= 100. "active" if last_viewed within 7 days. Otherwise "viewed"
   - Build investor_label: if investor_email exists use it, else if share_token_id exists use `"Anonymous (...{token_last4})"` by querying share_tokens.token, else use user email from users table
   - Sort: hot investors first, then active, then viewed. Within tiers sort by last_viewed descending
   - Return `AnalyticsSummaryResponse(investors=[...])`

3. **GET /analytics/investor/{investor_key}** (requires `get_current_user`) -- returns `InvestorDetail`:
   - Determine if investor_key is a user_id or share_token_id by checking both tables
   - Query analytics_events WHERE event_type='section_time' for this investor, aggregate SUM(duration_ms) GROUP BY section_id
   - Query queries table for this investor's questions (match by user_id or share_token_id)
   - Return `InvestorDetail(investor_key=..., sections=[...], questions=[...])`

4. **GET /analytics/new-view-count** (requires `get_current_user`) -- returns `NewViewCountResponse`:
   - Count distinct (user_id or share_token_id) in analytics_events WHERE founder_id = current user AND event_type = 'page_open' AND created_at > last_seen_timestamp
   - The last_seen_timestamp comes from a query param `?since=ISO_TIMESTAMP` (the frontend tracks when the user last visited the Analytics tab)
   - Return `NewViewCountResponse(count=N)`

5. **WebSocket /analytics/founder-notifications** -- founder notification channel:
   - Authenticate via `access_token` query param (same JWT validation pattern as query.py WebSocket)
   - Extract `sub` (user_id) from JWT payload as `founder_id`
   - Register in `_founder_connections` dict (new dict in notifications.py)
   - Keep alive with `await websocket.receive_text()`
   - On disconnect, unregister
