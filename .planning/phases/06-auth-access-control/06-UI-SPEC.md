## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.6 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 24px | 600 (semibold) | 1.25 |
| Display | 32px | 600 (semibold) | 1.2 |

Notes:
- Two weights only: 400 (regular) and 600 (semibold). Consistent with all prior phases.
- Display is used for auth page titles ("Sign in" / "Create account").
- Heading is used for dashboard Access section title ("Shared Access").
- Body is used for form field inputs, share link descriptions, and access expired page message.
- Label is used for form field labels, "or continue with" divider, link expiry timestamps, investor email addresses in the access table, and status text.

## Interaction Contract

### Login Flow

- Email and password fields with client-side validation (required, email format, min 8 chars for password)
- Submit button shows loading spinner (replace text with Loader2 icon spinning) during auth request
- On success: redirect to /dashboard (founders) or /pitch (investors) with success toast "Signed in"
- On failure: inline error message below form (not toast) -- "Invalid email or password"
- "Forgot password?" link below password field (navigates to /reset-password or triggers email)

### Signup Flow

- Email, password, confirm password fields
- Password mismatch shows inline error below confirm field: "Passwords do not match"
- Submit shows loading spinner during registration
- On success: redirect to /dashboard with success toast "Account created"
- On failure: inline error below form -- "An account with this email already exists" or generic error

### OAuth Flow

- Click OAuth button: redirect to provider
- On return: automatic session creation, redirect to appropriate landing page per role
- On failure: redirect to /login with error toast "Authentication failed. Please try again."

### Share Link Generation

- Click "Generate Link": creates token, displays copy-ready URL in an input-like readonly field
- Auto-copy to clipboard on generation with toast "Link copied to clipboard"
- Expiry selector: dropdown with options 7 days, 14 days, 30 days (default: 14 days)
- Generated URL format displayed: `https://domain/pitch?token=xxxxx`

### Email Invite

- Enter email, click "Send Invite": triggers email delivery, row appears in access table as "Invited"
- On success: toast "Invitation sent to {email}"
- On failure: toast "Could not send invitation. Please try again."
- Duplicate email detection: inline error "This email has already been invited"

### Access Revocation

- Click revoke button (X icon or "Revoke" text) on table row
- AlertDialog confirmation: "Revoke access? {investor identifier} will immediately lose access to the pitch."
- Confirm button: destructive variant, label "Revoke Access"
- Cancel button: ghost variant, label "Cancel"
- On confirm: row status changes to "Revoked" badge, revoke button disappears
- Toast: "Access revoked"

### Role-Based Route Protection

- middleware.ts intercepts all routes
- Unauthenticated user hitting protected route: redirect to /login with return URL preserved
- Investor hitting /dashboard or /documents: redirect to /pitch
- Token-link user accessing /pitch?token=valid: no auth required, read-only pitch access
- Token-link user accessing /pitch?token=invalid: render AccessExpiredPage

### SiteNav Role Awareness

- On load: SiteNav reads user session and role
- Founder: full nav (Pitch, Documents, Dashboard with PendingCountBadge) + UserAvatarMenu
- Investor: Pitch only + UserAvatarMenu (with Sign out only)
- Anonymous (token link): Pitch only, no account UI
- No auth (public pages): no SiteNav rendered

### Logout

- Click "Sign out" in avatar dropdown menu
- Clear session, redirect to /login
- Toast: "Signed out"

### Loading States

- Auth pages: submit button shows Loader2 spinning icon, inputs disabled during submission
- Access table: skeleton rows (3) matching table row height while loading
- SiteNav: brief skeleton for avatar area while session loads (prevent layout shift)

### Error States

- Network error on auth: inline form error "Unable to connect. Please check your connection."
- Session expired during navigation: redirect to /login with toast "Session expired. Please sign in again."
- Access table load failure: inline error within card "Unable to load access list. Please refresh."
