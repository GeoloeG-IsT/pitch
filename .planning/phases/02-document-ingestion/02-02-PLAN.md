**1. Create `apps/api/app/services/ingestion.py`** -- the orchestrator:

```python
import logging
from app.core.supabase import get_service_client
from app.services.parsers.pdf_parser import parse_pdf
from app.services.parsers.markdown_parser import parse_markdown
from app.services.parsers.excel_parser import parse_excel
from app.services.chunker import chunk_sections
from app.services.embedder import embed_chunks

logger = logging.getLogger(__name__)

PARSERS = {
    "pdf": parse_pdf,
    "md": parse_markdown,
    "txt": parse_markdown,  # Same heading-based parser for .txt
    "xlsx": parse_excel,
}

async def process_document(doc_id: str, file_bytes: bytes, file_type: str) -> None:
    client = get_service_client()
    try:
        # Update status to processing
        client.table("documents").update({"status": "processing"}).eq("id", doc_id).execute()

        # Parse into sections
        parser = PARSERS.get(file_type)
        if parser is None:
            raise ValueError(f"Unsupported file type: {file_type}")
        sections = parser(file_bytes)

        if not sections:
            raise ValueError(f"No content extracted from document (file_type={file_type})")

        # Chunk large sections (skip for Excel -- already sheet-per-chunk)
        if file_type != "xlsx":
            chunks = chunk_sections(sections, max_tokens=500, overlap_tokens=50)
        else:
            chunks = sections

        # Generate embeddings
        embedded = await embed_chunks(chunks)

        # Store chunks in database
        for chunk in embedded:
            client.table("chunks").insert({
                "document_id": doc_id,
                "content": chunk.content,
                "embedding": chunk.embedding,
                "section_number": chunk.section_number,
                "page_number": chunk.page_number,
                "chunk_type": chunk.chunk_type,
                "metadata": chunk.metadata,
                "token_count": chunk.token_count,
            }).execute()

        # Update document status to ready
        client.table("documents").update({
            "status": "ready",
            "metadata": {"chunk_count": len(embedded)},
        }).eq("id", doc_id).execute()

        logger.info(f"Document {doc_id} processed: {len(embedded)} chunks")

    except Exception as e:
        logger.error(f"Document {doc_id} processing failed: {e}")
        client.table("documents").update({
            "status": "error",
            "metadata": {"error": str(e)},
        }).eq("id", doc_id).execute()
```

**2. Create `apps/api/app/api/v1/documents.py`** -- REST endpoints:

Endpoints:
- `POST /api/v1/documents` -- Upload a document. Accepts multipart file + optional `title` query param. Returns `{id, status: "pending"}`.
- `GET /api/v1/documents` -- List all documents. Returns `{documents: [...], total: N}`. Each document includes chunk_count from chunks table.
- `GET /api/v1/documents/{doc_id}` -- Get single document with chunk_count.
- `DELETE /api/v1/documents/{doc_id}` -- Delete document (chunks cascade via FK).
- `PUT /api/v1/documents/{doc_id}` -- Re-upload: delete old chunks, update document, re-process.

Key implementation details:
- Read file bytes BEFORE dispatching background task (UploadFile closes after endpoint returns)
- Use `get_service_client()` for all DB operations (service role bypasses RLS)
- Detect file_type from filename extension: `.pdf` -> "pdf", `.xlsx` -> "xlsx", `.md` -> "md", `.txt` -> "txt"
- For upload, use a hardcoded demo user_id = "00000000-0000-0000-0000-000000000000" (auth comes in Phase 6). Add a `# TODO: Replace with auth context in Phase 6` comment.
- For the list endpoint, query chunk count with a separate count query: `client.table("chunks").select("id", count="exact").eq("document_id", doc_id).execute()`
- For delete, explicitly delete chunks first, then document (even though FK CASCADE exists, be explicit for clarity)
- For re-upload, delete old chunks, update document record (status=pending, new file_name, file_size_bytes), then dispatch background task
- File upload to Supabase Storage: store at path `documents/{doc_id}/{file_name}`. Create the storage bucket "documents" if it doesn't exist (catch 409 if already exists).
- Use `uuid.uuid4()` to generate doc_id before insert so we can use it in storage path.
