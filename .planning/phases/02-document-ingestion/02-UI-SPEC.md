## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` oklch(1 0 0) | Page background, main content area |
| Secondary (30%) | `var(--card)` oklch(1 0 0) / `var(--muted)` oklch(0.97 0 0) | Document cards, upload drop zone background, status badges background |
| Accent (10%) | `var(--primary)` oklch(0.205 0 0) | Upload CTA button, active document count indicator, progress bar fill |
| Destructive | `var(--destructive)` oklch(0.577 0.245 27.325) | Delete document button, error status badge |

Accent reserved for: primary "Upload Documents" CTA button, upload progress bar fill, active processing count badge. Never applied to document cards, status text, or navigation elements.

Semantic status colors for document processing states:

| Status | Color | Element |
|--------|-------|---------|
| Pending | `var(--muted-foreground)` | Status badge text, muted clock icon |
| Processing | `var(--chart-1)` oklch(0.809 0.105 251.813) | Animated progress bar, spinner icon |
| Ready | `hsl(142 71% 45%)` (green-500 equivalent) | Status badge, checkmark icon |
| Error | `var(--destructive)` | Status badge, error icon, retry link |

Source: `globals.css` tokens. Green for "ready" follows Phase 1 health status badge pattern (`bg-green-500`).

## Interaction Contracts

### Upload Flow

1. **Drop zone**: Full-width dashed border card at top of document list. Accepts drag-and-drop or click-to-browse. Dashed border uses `var(--border)`, on drag-over border becomes `var(--primary)` with `var(--muted)` background fill.
2. **File selection**: Native file picker filtered to `.pdf, .xlsx, .md, .txt`. Multiple file selection enabled.
3. **Immediate feedback**: On file selection, a new document card appears at top of list with status "Queued" and a progress bar at 0%.
4. **Processing**: Progress bar animates (indeterminate pulse, since backend processing time is unknown). Status updates to "Indexing...".
5. **Completion**: Progress bar fills to 100%, status changes to "Ready", toast notification appears. Card shows chunk count ("{n} sections indexed").
6. **Error**: Progress bar turns destructive color, status changes to "Failed", toast notification with error message. Card shows "Retry" link.

### Document List

1. **Layout**: Vertical stack of document cards, 8px gap between cards.
2. **Card content**: File icon (by type: PDF, Excel, text), document title, file size, upload date, status badge, chunk count (if ready), overflow menu button (three dots).
3. **Overflow menu**: "Replace File" and "Delete" actions. Delete is styled in destructive color.
4. **Empty state**: Centered illustration-free layout with heading, body text, and format hint. Upload drop zone doubles as the empty state CTA.
5. **Loading state**: 3 skeleton cards matching document card dimensions.

### Delete Flow

1. **Trigger**: "Delete" from overflow menu.
2. **Confirmation**: Dialog with heading, body, "Cancel" (secondary) and "Delete Document" (destructive) buttons.
3. **Execution**: On confirm, card fades out, toast confirms deletion.

### Re-upload Flow

1. **Trigger**: "Replace File" from overflow menu opens file picker.
2. **Confirmation**: Dialog warns that existing index will be replaced.
3. **Execution**: On confirm, card status resets to "Queued", processing restarts.

### Status Polling

1. **Mechanism**: After upload, poll `GET /api/v1/documents/{id}` every 3 seconds until status is "ready" or "error".
2. **Timeout**: Stop polling after 5 minutes, show error state with "Processing is taking longer than expected. Refresh to check status."
